# Organisation du dépôt GitHub

Tous les travaux doivent être regroupés dans un seul dépôt GitHub en respectant l'arborescence suivante :

```text
.
├── dart
│   ├── exercices
│   │   ├── exercice_01
│   │   ├── exercice_02
│   │   └── ...
│   ├── projets
│   │   └── nom_du_projet
│   └── tutorial
│       ├── chapitre_01
│       ├── chapitre_02
│       └── ...
│
└── flutter
    ├── exercices
    │   ├── exercice_01
    │   ├── exercice_02
    │   └── ...
    ├── projets
    │   ├── projet_dribbble
    │   └── autre_projet
    └── tutorial
        ├── chapitre_01
        ├── chapitre_02
        └── ...
```

## Description des dossiers

### `dart/`

Contient tous les travaux réalisés en langage Dart.

#### `dart/exercices/`

Contient les exercices d'algorithmique et de programmation Dart.

#### `dart/projets/`

Contient les projets réalisés uniquement avec Dart.

#### `dart/tutorial/`

Contient les exercices, exemples et travaux pratiques réalisés pendant les tutoriels et séances de formation.

---

### `flutter/`

Contient tous les travaux réalisés avec Flutter.

#### `flutter/exercices/`

Contient les exercices Flutter réalisés en classe ou à domicile.

#### `flutter/projets/`

Contient les projets Flutter complets, notamment le projet de reproduction d'une maquette Dribbble.

#### `flutter/tutorial/`

Contient les exemples, démonstrations et mini-projets réalisés pendant les séances de tutoriels.

---

## Règles de nommage

* Utiliser des noms de dossiers explicites.
* Éviter les espaces dans les noms de fichiers et dossiers.
* Utiliser de préférence le format `snake_case`.

Exemples :

```text
exercice_05
table_multiplication
gestion_notes
projet_dribbble
application_ecommerce
```

---

## README principal

Le fichier `README.md` à la racine du dépôt doit obligatoirement contenir :

1. Le nom de l'étudiant ;
2. La filière et le niveau ;
3. La description de l'organisation du dépôt ;
4. Les instructions pour exécuter les projets ;
5. Les liens vers les projets réalisés.

Tout dépôt ne respectant pas cette organisation pourra faire l'objet d'une pénalité lors de l'évaluation.
